OVERVIEW
--------
Extends expt2 along three dimensions:

  1. Wider token sweep    — adds 16k/32k/65k tokens to find the compute-bound
                            turnover and connect to Expt 1's 65k data points.

  2. NVLS × P2P ablation — 4-cell factorial ({NVLS on/off} × {P2P on/off}) to
                            attribute slowdown to NVLink-SHARP loss vs. P2P-IPC
                            loss independently.

  3. Bandwidth rungs     — vary NCCL_MAX_NCHANNELS 1→8 at fixed PCIe transport
                            for a dose–response curve (bandwidth-bound test).

  4. Routing imbalance   — 6 named routing modes (uniform → worst-case) swept
                            against NVLink vs. PCIe to test whether skew
                            amplifies or shifts interconnect sensitivity.

  Note on achieved bandwidth (item #3 analysis):
    allgather_recv_bytes and bucket_max_rank_network_ms are already logged in
    results.csv. Achieved GB/s = allgather_recv_bytes / (network_ms × 1e-3 × 1e9).
    No code changes needed — this is a post-processing / notebook step.

Shape: Qwen3-30B-A3B (hidden=2048, inter=768, E=128, topk=8, bf16)
Backend: allgather_reducescatter
Routing: see per-study below
Warmup: 30 iters | Measure: 100 iters | Seed: 42
Hardware: PACE ICE H200 SXM5, single-node, 4 GPUs/node

Routing modes (6):
  uniform    : round-robin; α≈1.0
  zipfian    : P(expert k) ∝ 1/k; realistic MoE proxy; α≈2–6×

ENGINEERING NOTES (inherited from expt2)
================================================================================

NCCL P2P / SLURM --gpus-per-task incompatibility (resolved in expt2, 2026-06-27):
  NCCL 2.30 selects P2P/CUMEM (or P2P/IPC) for intra-node transfers.
  Both require peer GPUs in CUDA_VISIBLE_DEVICES. SLURM --gpus-per-task=1
  restricts each rank to CUDA_VISIBLE_DEVICES=0 only, so peer import fails
  with "invalid device ordinal" / "invalid argument" → "unhandled cuda error".
  Fix: removed --gpus-per-task=1 from srun. All job-allocated GPUs stay visible;
  each rank calls torch.cuda.set_device(local_rank) in distributed.py.

Custom allreduce patch:
  vllm's CustomAllreduce calls can_device_access_peer(rank, world_size) using
  global rank as device ID, which breaks when only device 0 is visible. Patched
  to always return False in distributed.py (forces NCCL allreduce instead).

NCCL_NVLS_ENABLE=0 requirement on H100/H200:
  NCCL_P2P_DISABLE=1 alone may NOT disable NVSwitch on H100/H200 nodes.
  NCCL_NVLS_ENABLE=0 is required to disable the NVLink SHARP path. Verify
  with NCCL_DEBUG=INFO (logged to nccl_logs/) that the intended transport
  path is selected before trusting results from degraded-transport conditions.
